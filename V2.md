# mac-app-init v2

## 핵심 원칙

**TUI는 빈 껍데기다.** 도메인을 설치해야 비로소 뭔가를 할 수 있다.

```
v1:  TUI 탭이 하드코딩 → 설치 안 된 도메인도 탭이 보임
v2:  TUI 탭이 동적 → 설치된 도메인만 탭이 생김
```

## 레이아웃

가로 탭 대신 **세로 사이드바** (VS Code 스타일).

```
┌─ Sidebar ─┬─ Content ───────────────────────┐
│ Install   │                                 │
│ ───────   │                                 │
│ (설치된   │  선택한 항목의 UI               │
│  도메인이 │                                 │
│  여기에)  │                                 │
│           │                                 │
│           │                                 │
└───────────┴─────────────────────────────────┘
```

설치 0개일 때:
```
┌─ Sidebar ─┬─ Content ───────────────────────┐
│ Install   │                                 │
│           │  설치된 도메인이 없습니다.      │
│           │                                 │
│           │  Install에서 설치하세요.        │
└───────────┴─────────────────────────────────┘
```

## 도메인 ↔ TUI 통합

각 도메인 바이너리가 자기 TUI UI를 제공. 방식:

### `tui-spec` 명령

모든 도메인이 `tui-spec` 서브커맨드를 지원. JSON으로 UI 구조 반환.

```bash
mac-domain-keyboard tui-spec
```

```json
{
  "tab": { "label": "Keyboard", "icon": "⌨" },
  "sections": [
    {
      "title": "Status",
      "kind": "key-value",
      "items": [
        { "key": "Caps Lock → F18", "value": "✓ 적용됨", "status": "ok" },
        { "key": "LaunchAgent", "value": "✓ 등록됨", "status": "ok" }
      ]
    },
    {
      "title": "Actions",
      "kind": "buttons",
      "items": [
        { "label": "Setup", "command": "setup", "key": "s" },
        { "label": "Remove", "command": "remove", "key": "x" }
      ]
    }
  ]
}
```

TUI는 이 spec을 받아서 공통 렌더러로 그림.

### 액션 실행

```
TUI에서 "Setup" 버튼 클릭/Enter
→ mac-domain-keyboard setup 실행
→ 출력 캡처 → Output 패널에 표시
→ tui-spec 다시 조회 → UI 갱신
```

## 지원 위젯 타입

도메인이 사용할 수 있는 spec 위젯:

- `key-value` — 라벨/값 쌍 (Status 표시용)
- `table` — 헤더 + rows (목록용)
- `buttons` — 액션 버튼들
- `form` — 입력 폼 (Connect add 같은 것)
- `tree` — 트리 (Install 탭용)
- `text` — 단순 텍스트

## 공통 커맨드 규약

모든 도메인 바이너리는 다음을 지원해야 함:

```bash
<domain> --help            # 사람이 보는 도움말
<domain> tui-spec          # TUI용 JSON
<domain> <subcmd> [args]   # 실제 동작
```
